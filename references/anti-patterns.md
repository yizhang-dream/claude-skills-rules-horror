# 反套路指南与质量检查清单

## 一、七大常见误区

### 误区 1：规则堆砌成说明书
**表现**：开头甩十几条红色大字规则，没有节奏感，读者还没进入状态就被信息淹没。
**正确做法**：规则逐步展开。前几条看起来像正常规章制度，异常慢慢渗透。读者需要"发现"不对，而不是被"告知"不对。

### 误区 2：规则太多太满
**表现**：单份文档超过15条规则，读者审美疲劳，逻辑漏洞随规则数量指数增长。
**正确做法**：
- 单份文档 8-15 条规则
- 多文档总规则控制在 30 条以内
- 如果确实需要更多信息 → 拆分到不同文档/阵营

### 误区 3：过度解释
**表现**：
- "它"被详细描述了外貌、习性、来源
- 规则后面附"真相解析"
- 每个超自然现象都有"因为..."
**正确做法**：永远不要解释"为什么"。只给"怎么做"或"不要怎么做"。让读者自己推导。

### 误区 4：依赖血腥/怪物
**表现**：写"地上全是血""门后有腐烂的尸体""怪物有三只眼睛"
**正确做法**：最恐怖的描写是暗示——
- 不写"地上全是血"，写"地面最近做了深色重新铺设，不要询问铺设材料的来源"
- 不写"怪物"，写"你听到的声音可能来自你认识的人，但不要回应"

### 误区 5：情绪过载无缓冲
**表现**：每条规则都是生死攸关，从头紧张到尾。
**正确做法**：穿插"安全区"和日常细节。读者需要呼吸空间。
- 前几条规则可以完全正常（如"保持宿舍卫生""按时熄灯"）
- 规则的恐怖程度呈波浪式起伏，而非直线上升

### 误区 6：内部逻辑崩坏
**表现**：
- 规则A说某时间是安全的，规则B在同一时间安排了危险事件
- 某个设定在前期是关键威胁，后期被完全遗忘
- 阵营之间传播信息的方式没有解释就突然改变
**正确做法**：Step 4 逻辑校验必须逐条执行，不能跳过。

### 误区 7：AI 生成套路
**表现**：
- 规则总以"最后，请记住..."结尾
- 大量使用"并""且""以及""值得一提的是""需要注意的是"
- 每条规则都是"如果...请..."的标准句式，毫无变化
- 最后一段总是温馨/励志/感人收尾
**正确做法**：Step 6 语言润色必须严格执行。规则结尾应该戛然而止或留下一句让人不安的话。

### 误区 8：直白陈述异常本质
**表现**：
- "雾是活的""它不是人""那扇门通往另一个世界""他们已经死了"
- 即使放在最知情的内部文档（纸条/日记/幸存者留言）中也是错误的
- 以为"让角色说出来"就等于"没直接告诉读者"
**正确做法**：所有文档——无论知情度多高——都只描述行为，不陈述本质。
- ✗ "雾是活的" → ✓ "雾会等你。你不出门它就在窗外挂着。你出门，三分钟内到你面前。"
- ✗ "那不是门" → ✓ "走廊尽头绿色那扇——不要碰。走到它面前时闭眼。如果听到它开了，跑。"
- ✗ "他们已经不是人了" → ✓ "注意同事的说话节奏。如果他的断句方式和昨天不一样，换班。"

### 误区 9：万能保护物品
**表现**：某个物品（铁器/盐/符纸/特定颜色衣服）在所有规则中都是无条件解决方案，"握紧铁器就行了"。
**正确做法**：保护物品必须有代价或边界条件——
- 铁器有效，但握太久手会失去温度，且对已经违反过规则X的人无效
- 白狮子能保护你，但你不能确定看到的是白狮子还是别的什么
- 保护物品本身可能被污染，需要定期更换/检查

### 误区 10：抽象概念节点
**表现**：核心概念节点是抽象分类、编码、或不可触摸的"现象"。
- ✗ "B849.1类索书号""异常""规则本身""频率""信号"
- 读者无法在脑海中形成对这些东西的感官印象
**正确做法**：所有核心概念节点必须是物理对象/物质——可触摸、可摄入、可踩踏、可被触碰。
- ✓ "兔子血""瓷砖缝""白手套上的灰斑""水""铁器""声音"
- 《动物园》恐怖来自你看到兔子时的生理反应，不是因为"分类学意义被污染了"

### 误区 11：恐怖天花板太低
**表现**：隐藏设定中最坏的结果是"看到可怕的东西""感到不安""做噩梦"。
**正确做法**：规则怪谈的终极恐惧必须涉及——
- 身份被替代（你不知道自己还是不是自己）
- 永久困住（你出不去了，而且可能已经困了很久而不自知）
- 认知不可逆损毁（你再也无法相信自己的感官）
- 被置换（你身边的人已经不是原来那个人了）
- 如果读者推完所有线索后只是觉得"有点诡异"→ 恐怖天花板不够，重做 Step 2

### 误区 12：实体是传统物理威胁
**表现**：异常实体通过物理接触/空间接近来"攻击"受害者。规则是纯防御性的。
**正确做法**：首选"认知即污染"机制——
- 了解/理解/认识这个实体本身就是感染途径
- 每多读一条规则、每多理解一层真相，你就越脆弱
- 这创造了规则怪谈独有的悖论：规则保护你，但读规则让你更危险
- 《动物园》之所以不可超越，核心就在"知道即污染"——读者越分析越怕
- 即使不用认知机制，至少要让规则的"否定性指令"本身成为心理陷阱

### 误区 13：文档"叙事化"
**表现**：规则文件读起来像微型小说——有时间线、人物弧光、情感高潮。
**正确做法**：规则怪谈的文档是"被发现的物件"（found documents），不是故事——
- 物业值班记录应该像真实的行政日志，大量无聊条目中偶尔出现异常
- 便利贴是匆忙写下的提醒，不是写给读者看的情节推进
- 每份文档有自己独立的、受限的视角，不知道其他文档说了什么
- 文档作者不应该"恰好"把所有信息组织成一篇完整叙事

### 误区 14：陈述违反规则的后果
**表现**："如果不这样做，你就会...""违反此条将导致...""以免..."
**正确做法**：永远不说违反规则会怎样。读者应该永远不知道——
- ✗ "握紧铁器，否则你会看到不该看的东西"
- ✓ "握紧墙上的铁质装饰品。等冲动过去。天亮后再做决定。"
- 《动物园》从不说进入海洋馆会怎样——只说不要进去
- 未知的后果 >>>> 任何可以被描述的惩罚

### 误区 15：公文"请"字泛滥
**表现**：每条规则都以"请"开头——"请勿在走廊奔跑""请保持安静""请注意"
**正确做法**：真实中文公文很少用"请"。每5条规则最多1个"请"——
- ✗ "请勿在夜间进入该区域，请注意安全"
- ✓ "禁止夜间进入该区域。"
- ✓ "本区域22:00后关闭。"
- 用"须/应/不得/禁止/严禁/勿"替代"请"

### 误区 16：字条写成编号列表
**表现**：便利贴/纸条上出现整洁的"1. 2. 3. 4."编号，每个条目完整的"主题句+解释"
**正确做法**：真人匆忙写的字条是碎片的、跳跃的——
- ✗ "1. 黄色价签不是店长放的。2. 外面下雨那条是真的。3. 冷藏柜..."
- ✓ "黄的价签——不是店长放的。你别管谁放的。反正别碰。
      对了还有那个下雨的事。是真的。
      冷藏柜——算了这个你自己看吧我写不动了"
- 字条可以：半截句子、重复、涂改、突然换话题

### 误区 17：文本太干净
**表现**：所有文件完整无缺、字迹工整、没有涂改、没有水渍、没有撕页
**正确做法**：真实"被发现的文件"应该有缺陷——
- 被水浸模糊的半行字（用省略号或■替代）
- 被撕去的页面痕迹（"以下两页已缺失"）
- 涂改——~~划掉~~后改写
- 字迹逐渐潦草（后期字条与前期明显不同）
- 日期缺失或矛盾
- 每份文档最多一处缺陷。不要过度。

### 误区 18：句式千篇一律
**表现**：所有规则都是"如X，请Y"模式：如发现...请联系...如遇到...请立即...如听到...请不要...
**正确做法**：混用五种句式——祈使短句/条件+指令/纯粹禁令/定义+限制/无主语句。同一句式不得连续出现3次以上。读一遍你所有的规则。如果超过3条以相同句式开头→重写其中2条。

### 误区 19：句子太长
**表现**：两个标点之间连续文字超过20字。读者一口气读不完，气口不对。
**正确做法**：15-20字断句法。超了就加标点或拆句。短句(≤10字)制造紧张，长句(15-20字)承载信息。相邻句长短相差 >5字。
- ✗ "如发现瓶装饮料瓶身水珠在室温下持续五分钟以上未蒸发"（23字）
- ✓ "瓶身水珠在室温下超过五分钟不蒸发——该瓶移至冷藏柜底层。"

### 误区 20：翻译腔三件套
**表现**：滥用"的"/弱动词/"不是...而是..."
- 连续两个"的"→ 删一个。连续三个"的"→ 拆句重写
- "进行清点""加以记录""实施管理""作出决定"→ 删掉弱动词
- "不是A而是B"→ 直接说B
**正确做法**：
- ✗ "对退货篮中的商品进行每日清点并加以记录"
- ✓ "每日清点退货篮商品。记录数量。"
- ✗ "黄色价签不是由普通员工设置，而是由店长指定的"
- ✓ "黄色价签由店长指定。"

### 误区 21：缺行业真实用语
**表现**：公文读起来像"翻译成中文的外国文件"。
**正确做法**：每个场景有自己的术语体系——
- 便利店：唱收唱付、先进先出、理货、排面、盘点、报损
- 物业：报修、巡查、台账、责令、限期整改
- 图书馆：编目、上架、剔旧、索书号、馆藏
- 如果不知道这个行业的术语→搜一下真实的员工手册/物业通知/图书馆须知

## 二、烂俗桥段黑名单

以下桥段已被无数作品用烂，创作时必须避免（除非你能做出真正新颖的反转）：

1. **"第X条规则不存在"** — 已经被用到失去任何恐怖感
2. **"不要回头"** — 太泛，没有任何信息量
3. **"他们/它们不是人"** — 直接把恐怖说出来了，破坏留白
4. **"如果你看到了这条规则，说明你已经..."** — 老套
5. **午夜/凌晨3点是特殊时间** — 可以用但需要新颖的具体时间（如3:17、2:22）
6. **镜子里看不到自己** — 太常见
7. **所有人其实都已经死了** — 老套的终极反转
8. **结尾"快跑""活下去""不要相信任何人"** — 情感宣泄，消解了冷静的恐怖
9. **规则最后一条是"不要遵守以上任何规则"** — 不负责任的偷懒写法
10. **动辄"立刻离开""立刻逃跑"** — 好的规则怪谈中，异常是可以回避的，规则是用来保护你的

## 三、质量检查清单

Step 5 恐怖打磨时逐项检查：

### 恐怖天花板（一票否决）
- [ ] 读者推完所有线索后，最坏结果是否涉及身份替代/永久困住/认知不可逆损毁/被置换？
- [ ] 如果不是 → 回到 Step 2 重新设计隐藏设定

### 认知即污染（一票否决）
- [ ] 污染/危险机制是否与认知行为绑定（看到/读到/理解/认出）？
- [ ] 是否存在"知道越多越危险"的悖论？
- [ ] 如果不是 → 考虑将认知作为感染途径

### 概念节点物理性 + 符号体系（一票否决）
- [ ] 所有核心概念节点是否都是物理对象/物质？
- [ ] 每个概念节点是否对应污染进程中的特定阶段/状态？（符号体系）
- [ ] 是否存在抽象编码/分类/概念作为节点 → 替换为对应物理实体

### 恐怖感
- [ ] 是否有 ≥3 处需要读者自己推理才能体会到的恐怖点？（不是文本直接说的）
- [ ] 是否有一处让读者产生"等等，那如果...的话岂不是..."的自我追问？
- [ ] 读完全文后是否会产生"回看前面，细思极恐"的效果？

### 语言（新增）
- [ ] 官方文档是否使用了公文惯用语（本/须/应/不得/经/予以）？
- [ ] "请"字每5条规则是否≤1次？
- [ ] 是否有≥3种不同句式（祈使短句/条件+指令/纯粹禁令/定义+限制/无主语句）？
- [ ] 字条类文档是否碎片化、无编号、口语化？
- [ ] 是否至少有一处文本"不完美"（水渍/涂改/撕页/字迹变化）？
- [ ] AI连接词是否全部删除？结尾是否戛然而止？
- [ ] 是否有至少一个规则让读者觉得"这是我想不到的设定"？
- [ ] 黑名单中的桥段是否全部避开？

### 逻辑
- [ ] 所有规则矛盾是否在隐藏设定卡中有合理解释？
- [ ] 规则推导是否可追溯（每条规则 ← 对应设定）？
- [ ] 多条规则之间是否存在隐含的因果关系？（而非孤立列表）
- [ ] 保护物品是否有明确的局限性？是否存在"万能解药"物品需要修正？

### 留白
- [ ] ≥30% 的关键信息处于 L2 或 L3 披露层级？
- [ ] 是否存在读者可以自行拼凑但文本从未明说的"隐藏故事"？
- [ ] "它"或等价的异常实体是否从未被直接描述？
- [ ] 所有文档——包括最知情的纸条/日记——是否都是行为描述而非本质陈述？（"雾会等你" not "雾是活的"）

### 语言
- [ ] 是否使用冷静、官僚、公文式的口吻？
- [ ] 是否存在 AI 连接词（值得一提的是、综上所述、并、且、以及）？
- [ ] 规则节奏是否有变化（短句/长句交替）？
- [ ] 结尾是否避免了温馨/感人/励志/总结性的收尾？

### 耦合
- [ ] 核心概念节点是否 ≤4 个？
- [ ] 是否有 ≥60% 的规则涉及核心概念节点？
- [ ] 概念节点之间是否存在明确的关联关系（而非各自孤立）？
- [ ] 是否存在"孤儿规则"——某条规则不与任何核心节点耦合？

### 信息增量（多文档时）
- [ ] 每份文档是否贡献了其他文档没有的新信息？
- [ ] 是否存在某份文档只是复读/加强已有信息？
- [ ] 不同文档之间是否存在真正的矛盾（而非只有知情度递增）？
